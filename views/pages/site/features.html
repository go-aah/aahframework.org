Features

Summary of aah features and capabilities that empowers engineering and operation/devops teams.

  • Detailed documentation of every features, options and configurations. Its getting better and better
  • Perfect fit for Web applications and/or RESTful APIs
  • Flexible configuration format, very similar HOCON syntax
  • Easy & elegant way to implement powerful Application Security. Provides capabilities to prevent OWASP top 10 web application security risks
  • Internalization and Localization with zero coding effort, just configuration and convention
  • Easy maintenance & debugging - Server Access Log, Server Dump Log
  • Go idiomatic, intuitive API, typical naming conventions

List of features and capabilities provided by aah framework

  • Server and Extension Points
  • Configuration
  • URL Routing and Reverse Route
  • Request Parameters - Auto Parse and Bind  Since v0.8
  • Security - Authentication and Authorization  Since v0.7
  • Security - Session Management
  • Security - Anti-CSRF  Since v0.9
  • View Engine
  • i18n Internalization and Localization
  • Middleware
  • Interceptors
  • Secure HTTP Headers  Since v0.8
  • Reply Builder
  • Event Publisher/Emitter
  • Content Negotiation  Since v0.8
  • Centralized Error Handling  Since v0.8
  • Static File Delivery
  • Logger
  • HTML Minify  Since v0.6
  • Easy to use Application Binary
  • Essentials Library
  • Hot-Reload for Development  Since v0.7
  • Upcoming Features, refer to Roadmap
