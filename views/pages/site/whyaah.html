Why aah?

aah aims to provide necessary components to build modern Web and API application with secure, high performance, scalable and yet lightweight, flexible. aah takes care of infrastructure, boilerplate code, repetitive activities, reusable components, etc.  aah is not a micro web framework.


Have a look at features to know your benefits of using aah and its very well documented.

  •   Security aware framework, secure session, CSRF prevention, XSS prevention, authentication, authorization
  •   Extensible at every module/feature level. If you're constrained, immediately raise an issue at GitHub.
  •   Provides proven project structure and organization of application pieces.
  •   Highly maintainable, it grows along with your application use case.
  •   Gradually getting mature and feature sets are getting added release by release.

aah Go web framework's initial stable version 0.5 released on May 19, 2017.
